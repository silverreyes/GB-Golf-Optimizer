Create the `deploy/` directory at project root with three files.

**File 1: deploy/gbgolf.service**

systemd service unit. Use placeholder `<deploy_user>` and `/path/to/GBGolfOptimizer` that the user replaces during deployment:

```ini
[Unit]
Description=GB Golf Optimizer (Gunicorn)
After=network.target

[Service]
User=<deploy_user>
Group=www-data
WorkingDirectory=/path/to/GBGolfOptimizer
Environment="PATH=/path/to/GBGolfOptimizer/venv/bin"
Environment="SCRIPT_NAME=/golf"
ExecStart=/path/to/GBGolfOptimizer/venv/bin/gunicorn \
    --workers 2 \
    --bind unix:/path/to/GBGolfOptimizer/gbgolf.sock \
    --umask 007 \
    wsgi:app
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

2 workers: The Hostinger KVM 2 has 2 vCPUs; optimization is CPU-bound so 2 sync workers avoids core contention on simultaneous requests.

**File 2: deploy/gameblazers.silverreyes.net.nginx**

Nginx server block for the subdomain. The `/golf` prefix is passed intact to Gunicorn (no trailing slash after the socket in proxy_pass) so Flask sees it as SCRIPT_NAME, not a path component:

```nginx
server {
    listen 80;
    server_name gameblazers.silverreyes.net;

    # GB Golf Optimizer at /golf
    # proxy_pass has NO trailing slash — full URI including /golf passes through to Gunicorn
    # SCRIPT_NAME=/golf in systemd env tells Flask/Werkzeug where the app is mounted
    location /golf {
        include proxy_params;
        proxy_pass http://unix:/path/to/GBGolfOptimizer/gbgolf.sock;
        proxy_set_header X-Forwarded-Prefix /golf;
    }

    # Future: /nfl location block for NFL optimizer
}
```

**File 3: deploy/DEPLOY.md**

Step-by-step deployment instructions. Include:

1. Prerequisites — Python >=3.10 installed (`python3 --version`), git, pip
2. Clone/upload the project to the VPS (e.g., `git clone` or `scp`)
3. Create a virtualenv and install: `python3 -m venv venv && source venv/bin/activate && pip install -e .`
4. Edit `deploy/gbgolf.service`: replace `<deploy_user>` with the VPS user, replace all `/path/to/GBGolfOptimizer` with the actual absolute path
5. Copy service file: `sudo cp deploy/gbgolf.service /etc/systemd/system/gbgolf.service`
6. Enable and start: `sudo systemctl daemon-reload && sudo systemctl enable gbgolf && sudo systemctl start gbgolf`
7. Check status: `sudo systemctl status gbgolf` — look for "active (running)"
8. Check that the socket was created: `ls -la /path/to/GBGolfOptimizer/gbgolf.sock`
9. Edit `deploy/gameblazers.silverreyes.net.nginx`: replace `/path/to/GBGolfOptimizer` with the actual path
10. Check existing Nginx config to avoid port conflicts: `sudo nginx -T | grep server_name` and `ss -tlnp` — confirm no other server block uses `gameblazers.silverreyes.net`
11. Copy Nginx config: `sudo cp deploy/gameblazers.silverreyes.net.nginx /etc/nginx/sites-available/gameblazers.silverreyes.net`
12. Enable site: `sudo ln -s /etc/nginx/sites-available/gameblazers.silverreyes.net /etc/nginx/sites-enabled/`
13. Test Nginx config: `sudo nginx -t`
14. Reload Nginx: `sudo systemctl reload nginx`
15. Smoke test: `curl -s -o /dev/null -w "%{http_code}" http://gameblazers.silverreyes.net/golf/` — expect 200
16. Optional: Add DNS A record for `gameblazers.silverreyes.net` pointing to VPS IP if not already set
17. Optional: Add Let's Encrypt SSL with `sudo certbot --nginx -d gameblazers.silverreyes.net`

Note about Open Claw coexistence: the Nginx config uses a separate `server_name gameblazers.silverreyes.net` block. It does NOT modify any existing server blocks. Verify with `sudo nginx -T` before reloading.
