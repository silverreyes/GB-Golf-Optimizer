<feature>
  <name>optimize() sequential orchestrator implementation</name>
  <files>gbgolf/optimizer/__init__.py, tests/test_optimizer.py</files>
  <behavior>
    Sequential solve algorithm:

    1. Create a mutable copy of the input pool: `pool = list(valid_cards)`
    2. Initialize: `lineups: dict[str, list] = {}`, `notices: list[str] = []`
    3. For each contest in contests (in order — Tips first):
       a. `contest_lineups = []`
       b. For n in range(1, contest.max_entries + 1):
          - Call `selected = _solve_one_lineup(pool, contest)`
          - If selected is None:
            notices.append(f"Could not build lineup {n} for {contest.name}")
            continue (do NOT break — try remaining lineup slots)
          - Else:
            Create `Lineup(contest=contest.name, cards=selected)` and append to contest_lineups
            Remove used cards from pool using object identity:
              `used_ids = {id(c) for c in selected}`
              `pool = [c for c in pool if id(c) not in used_ids]`
       c. `lineups[contest.name] = contest_lineups`
    4. Return `OptimizationResult(lineups=lineups, unused_cards=pool, infeasibility_notices=notices)`

    Critical correctness points:
    - Pool mutation: each successful solve immediately reduces the pool. Cards from Tips lineup 1 are NOT available for Tips lineup 2.
    - Infeasibility is not fatal: if lineup 2 fails, still try lineup 3. All partial results are returned.
    - Card identity: use `id(c)` — NOT equality comparison. Two Card objects with identical fields but different Python identities are different cards (defensive against duplicate data).
    - Intermediate Tee receives whatever pool remains after ALL Tips lineups are processed — not a fresh copy of valid_cards.
    - The contests list ordering in the call determines priority — if caller passes Tips first, Tips is solved first. The function itself is order-agnostic.

    Anti-patterns to avoid:
    - Do NOT create a fresh pool copy for each contest
    - Do NOT use `if selected is None: break` — continue to next lineup slot
    - Do NOT use card equality (==) for pool filtering — use id()
    - Do NOT raise on any infeasibility condition
  </behavior>
