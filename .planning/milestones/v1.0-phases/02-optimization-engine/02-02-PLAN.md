def _solve_one_lineup(cards: list, config: ContestConfig) -> list | None:
    """
    Selects exactly config.roster_size cards from `cards` to maximize
    total effective_value subject to salary and collection constraints.

    Returns:
        list[Card] — the selected cards if an optimal solution exists

    Never raises on infeasibility.
    """

<feature>
  <name>_solve_one_lineup ILP implementation</name>
  <files>gbgolf/optimizer/engine.py, tests/test_optimizer.py</files>
  <behavior>
    Correct ILP formulation — all constraints from the requirements:

    1. Binary variable per card: x[i] in {0, 1} for each card index i
    2. Objective: maximize sum(cards[i].effective_value * x[i] for all i)
    3. Roster size: sum(x[i] for all i) == config.roster_size (exactly)
    4. Salary floor: sum(cards[i].salary * x[i]) >= config.salary_min
    5. Salary cap: sum(cards[i].salary * x[i]) <= config.salary_max
    6. Collection limits (upper bounds only, never minimums):
       For each (collection_name, limit) in config.collection_limits.items():
         eligible_indices = [i for i, c in enumerate(cards) if c.collection == collection_name]
         if eligible_indices: sum(x[i] for i in eligible_indices) <= limit
       NOTE: A lineup with 0 Weekly Collection cards is LEGAL if the constraint only caps at 3.
    7. Same-player constraint (within-lineup uniqueness):
       For each unique player in cards:
         player_indices = [i for i, c in enumerate(cards) if c.player == player]
         if len(player_indices) > 1: sum(x[i] for i in player_indices) <= 1
    8. Solve: prob.solve(pulp.PULP_CBC_CMD(msg=0))  -- msg=0 suppresses CBC output
    9. Status check: use pulp.LpStatus[prob.status] == "Optimal" to confirm feasibility
       If not "Optimal": return None  (never raise)
    10. Extract result: [cards[i] for i in range(len(cards)) if x[i].varValue > 0.5]
        NOTE: use > 0.5 (not == 1) to handle CBC floating-point output

    Edge cases to handle:
    - Empty cards list: return None (sum constraint == roster_size is immediately infeasible)
    - Cards fewer than roster_size: return None (naturally infeasible)
    - Pool has only one player with 2 cards: same-player constraint limits to 1 — that's correct

    Anti-patterns to avoid (from RESEARCH.md):
    - Do NOT use varValue == 1 (use > 0.5)
    - Do NOT raise ValueError on infeasibility (return None)
    - Do NOT treat collection_limits as minimums (all are <= constraints)
    - Do NOT use msg=1 (always msg=0 to suppress CBC output in tests)
  </behavior>
