**GREEN phase: Create gbgolf/fetcher.py**

Create `gbgolf/fetcher.py` with these functions:

1. **`parse_datagolf_name(raw: str) -> str`**
   - If `","` in raw: split on first comma, return `"{parts[1].strip()} {parts[0].strip()}"`
   - Else: return `raw.strip()`
   - Handles "Last, First", "Last Jr., First" (suffix stays with last name part), no-comma passthrough

2. **`_DataGolfPlayerProjection` Pydantic model** (private, boundary only)
   - Fields: Use EXACT field names from discovery response sample
   - At minimum: `player_name: str` and the projected fantasy points field (name from discovery)
   - Use `model_config = ConfigDict(extra="ignore")` to silently drop fields we don't need (salary, ownership, etc.)

3. **`write_fetch_log(log_dir: str, line: str) -> None`**
   - `os.makedirs(log_dir, exist_ok=True)`
   - Timestamp: `datetime.now(UTC).strftime("%Y-%m-%d %H:%M:%S UTC")`
   - Appends `f"{timestamp} | {line}\n"` to `{log_dir}/fetch.log`

4. **`write_projections(session, tournament_name: str, tour: str, players: list[dict]) -> int`**
   - Enable FK pragma for SQLite: `session.execute(text("PRAGMA foreign_keys = ON"))` (no-op on PostgreSQL)
   - DELETE FROM fetches WHERE tournament_name = :tn AND tour = :tour (CASCADE deletes projections)
   - INSERT INTO fetches (tournament_name, fetched_at, player_count, source, tour) VALUES (...) RETURNING id
   - Capture fetch_id = result.scalar_one()
   - Loop: INSERT INTO projections (fetch_id, player_name, projected_score) VALUES (...) for each player
   - session.commit()
   - Return fetch_id
   - All queries use `db.session.execute(text(...))` pattern per project convention

5. **`run_fetch(log_dir: str = "logs") -> str`**
   - Load DATAGOLF_API_KEY from `os.environ` (already in env via load_dotenv in app factory)
   - Call `httpx.get("https://feeds.datagolf.com/preds/fantasy-projection-defaults", params={"tour": "pga", "site": "draftkings", "slate": "main", "file_format": "json", "key": api_key}, timeout=30.0)`
   - Wrap in try/except for `httpx.HTTPStatusError`, `httpx.ConnectError`, `httpx.TimeoutException`, `Exception`
   - On any HTTP/network error: call `write_fetch_log(log_dir, f"ERROR | {error_description} | existing data preserved")`, return error summary string
   - On success: parse JSON through list of `_DataGolfPlayerProjection.model_validate(item)` for each item
   - **GUARD: if len(players) < 30**: call `write_fetch_log(log_dir, f"ERROR | Only {len(players)} players returned (minimum 30) | existing data preserved")`, return error summary. Do NOT touch the database.
   - Extract tournament name from API response (check discovery sample for field name -- may be top-level `event_name` or similar; if not in response, use a fallback like the current week's event from a secondary field)
   - For each player: `display_name = parse_datagolf_name(player.player_name)`
   - Build players list: `[{"player_name": display_name, "projected_score": player.<points_field>}]`
   - Call `write_projections(db.session, tournament_name, "pga", players_list)`
   - Call `write_fetch_log(log_dir, f"OK | {tournament_name} | {len(players_list)} players | fetch_id={fetch_id}")`
   - Return success summary string

Run `pytest tests/test_fetcher.py -x -q` -- all tests should PASS (GREEN).
