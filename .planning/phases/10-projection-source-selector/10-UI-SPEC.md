## Typography

All values match existing stylesheet. No new type styles are introduced.

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 14px (0.86rem) | 400 | 1.5 | Prompt (--font-body) |
| Form heading | 10px (0.62rem) | 700 | 1.5 | JetBrains Mono (--font-mono) |
| Radio label | 12px (0.75rem) | 400 | 1.5 | JetBrains Mono (--font-mono) |
| Staleness label | 10px (0.62rem) | 400 | 1.5 | JetBrains Mono (--font-mono) |

Sources:
- Form heading: existing `.form-row-heading` uses `0.62rem / 700 / --font-mono`
- Radio label: RESEARCH.md CSS example specifies `0.75rem / --font-mono`
- Staleness label: aligned to 10px (0.62rem) to match form heading size; the original 0.65rem (10.4px) was only 0.4px different and visually indistinguishable. Using the same 0.62rem value as form headings gives a consistent small-text tier.

## Component Inventory

### Visual Hierarchy

Visual anchor: the radio row (`.source-selector`) is the primary new interactive element and anchors the user's eye within the source selector block. The staleness label is secondary informational text, rendered at smaller size and muted weight to recede behind the radio options.

### Source Selector Row

New `.form-row` at the top of the `<details id="upload-section">` form, before the Roster CSV row.

**Structure:**
```
.form-row
  .form-row-heading  "Projection Source"
  .source-selector
    .source-radio
      input[type="radio"][name="source_radio"][value="auto"]
      "Auto"
    .source-radio
      input[type="radio"][name="source_radio"][value="csv"]
      "Upload CSV"
  input[type="hidden"][name="projection_source"]#projection-source-input
  .staleness-label#staleness-label  (conditional)
```

**CSS for new classes:**

```css
.source-selector {
  display: flex;

.source-radio {
  font-family: var(--font-mono);
```

### Projections CSV Zone (existing, modified behavior)

The existing Projections CSV `.form-row` (containing the file-upload-zone) gets a wrapper `<div id="projections-zone">` for JS show/hide targeting.

| Source selected | `#projections-zone` display | `#projections` required | `#staleness-label` display |
|-----------------|---------------------------|------------------------|---------------------------|
| Auto | `none` | removed | visible |
| Upload CSV | block (default) | `required` | `none` |

## Interaction Contract

### Radio toggle behavior (vanilla JS)

1. On `change` event for any `input[name="source_radio"]`:
   - Set `#projection-source-input` value to the selected radio's value
   - If `auto`: hide `#projections-zone`, remove `required` from `#projections`, clear any selected file (`projInput.value = ''`), show `#staleness-label`
   - If `csv`: show `#projections-zone`, add `required` to `#projections`, hide `#staleness-label`

2. On page load:
   - If `db_has_projections` is true: Auto radio checked, CSV radio unchecked, `#projections-zone` hidden, staleness label visible
   - If `db_has_projections` is false: Auto radio disabled, CSV radio checked, `#projections-zone` visible, staleness label shows "No projections available yet"

3. Switching from Auto to CSV: clears previously selected file (clean state)

### Form submission

- Hidden input `projection_source` value (`auto` or `csv`) is submitted with the form
- Backend validates: if `projection_source=auto` but DB empty, returns error
- No AJAX -- full page form POST, matching existing pattern

### No new interactions on results page

- Re-optimize form is unchanged; it uses serialized card pool from first optimize
- Lock/exclude JS is unchanged
- Unmatched player sidebar renders identically regardless of projection source

## Copywriting Contract

| Element | Copy |
|---------|------|
| Source selector heading | `Projection Source` |
| Auto radio label | `Auto` |
| CSV radio label | `Upload CSV` |
| Staleness label (fresh, example) | `Arnold Palmer Invitational — fetched 3 days ago` |
| Staleness label (stale, example) | `Arnold Palmer Invitational — fetched 8 days ago` |
| Staleness label (today) | `Arnold Palmer Invitational — fetched today` |
| Staleness label (1 day) | `Arnold Palmer Invitational — fetched 1 day ago` |
| Staleness label (N days) | `[Tournament Name] — fetched [N] days ago` |
| Empty state message | `No projections available yet` |
| Error: auto with empty DB | `No projections available — please upload a CSV` |
| Primary CTA | `Generate Lineups` (unchanged from existing) |

### Staleness label pluralization rules

| `days_ago` value | Display text |
|------------------|-------------|
| 0 | `fetched today` |
| 1 | `fetched 1 day ago` |
| >= 2 | `fetched [N] days ago` |

### Template expression

```jinja2
{% if latest_fetch %}
  {{ latest_fetch.tournament_name }} &mdash;
  {% if latest_fetch.days_ago == 0 %}fetched today
  {% elif latest_fetch.days_ago == 1 %}fetched 1 day ago
  {% else %}fetched {{ latest_fetch.days_ago }} days ago
  {% endif %}
{% elif not db_has_projections %}
  No projections available yet
{% endif %}
```

## States Matrix

| State | Auto radio | CSV radio | Staleness label | Projections zone | Hidden input value |
|-------|-----------|-----------|-----------------|------------------|--------------------|
| DB has fresh projections (< 7 days) | checked | unchecked | visible, normal color | hidden | `auto` |
| DB has stale projections (>= 7 days) | checked | unchecked | visible, muted color | hidden | `auto` |
| DB empty (no fetches) | disabled | checked | "No projections available yet", muted | visible | `csv` |
| User switches to CSV | unchecked | checked | hidden | visible | `csv` |
| User switches to Auto | checked | unchecked | visible | hidden | `auto` |
| POST error (any source) | restored to pre-submit state via `db_has_projections` + `latest_fetch` | same | same | same | same |
