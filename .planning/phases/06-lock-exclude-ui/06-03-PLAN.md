Template variable provided by both routes (confirmed in Plan 02):
    locked_card_keys — Python set of 4-tuples: {(player, salary, multiplier, collection), ...}
    Pass set() (empty set) from index() POST; pass set(locked_cards) from reoptimize() POST.

IMPORTANT — Jinja2 set membership note from research:
    locked_card_keys is a Python set of tuples.
    Jinja2 `in` operator works correctly with Python sets.
    The template receives it as a set — O(1) lookup per row.
    Guard: `{% if locked_card_keys and (...) in locked_card_keys %}` — the `and` guard handles the case where locked_card_keys is None (shouldn't happen after Plan 02, but defensive).
